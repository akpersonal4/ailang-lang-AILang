# DX-005 Test Generator — Design Document

**Post-implementation reference. See actual code for current state.**

---

## Decision Record

| Decision | Chosen | Rationale |
|----------|--------|-----------|
| Test output directory | `tests/generated/` as Python package | Standard pytest discovery, no custom config |
| Test file naming | `test_<app>_generated.py` | Clearly identifiable as auto-generated |
| Generation approach | **Pure Python** (no `.tmpl` files) | Easier refactoring, testing, IDE support |
| Pipeline stages | **Discovery → Analysis → Generation** | Separation of concerns |
| Intermediate model | `TestCase` dataclass | Facts first, rendering second — extensible to MD/AI/docs |
| File header | `AUTO-GENERATED BY AILang DX-005` | Prevents confusion with handwritten tests |
| Overwrite protection | Never overwrite without `--force` | Prevents accidental data loss |
| CLI design | `--dry-run`, `--force`, `--app`, `--report-only`, `--quiet` | Future-extensible |

## Architecture

```
tools/ail_testgen/
├── __init__.py       — Public API
├── models.py         — TestCase, TestCategory, CoverageReport
├── discovery.py      — App discovery + existing test discovery
├── analyzer.py       — Coverage gap analysis
├── generator.py      — Pure Python test file generation
├── validator.py      — pytest-based test verification
├── reporter.py       — MD + JSON report generation
└── __main__.py       — CLI entry point
```

## Processing Pipeline

```
Repository
    ↓
Discovery (apps/*/main.ail, tests/*.py)
    ↓
Analysis (coverage gaps, missing tests)
    ↓
TestCase model
    ↓
Generator (pure Python, no templates)
    ↓
tests/generated/test_<app>_generated.py
    ↓
generated/TEST_GENERATION_REPORT.md + .json
```

## Design Principles

1. **Generate, don't overwrite** — `--force` required to overwrite existing
2. **Reuse before create** — `tools/common/` shared library
3. **Deterministic output** — same source always produces same tests
4. **Consistent naming** — `test_app_<name>_generated.py`
5. **No duplicate tests** — coverage analysis prevents redundant generation
6. **Human review before committing** — all generated files in `tests/generated/`
