# v1.1.0 Release Notes

**Release Date**: 2026-07-20

## Version

v1.1.0

## Improvements

### Lock file format update (M77.1)
- `ail.lock` now uses `[[package]]` (singular), `resolved_version` instead of `version`, and `commit` field for git dependencies
- Git+ URL shorthand: Manifest now supports `audit = "git+https://..."` string format alongside the existing table syntax
- Version conflict detection: Resolver now detects and reports version conflicts with actionable diagnostics showing conflicting constraints
- Improved circular dependency detection: Cycles detected even when involving already-visited packages, with clear cycle path in error message

### CLI enhancements
- `--verbose` flag: `ail install --verbose` shows manifest path, project root, lib/cache dirs, dependency details, and per-package install steps
- `--frozen` alias: `ail install --frozen` is a shortcut for `--frozen-lockfile`
- Lockfile staleness exit code: Returns exit code 4 (LOCKFILE_MISMATCH) instead of generic FAILURE when lockfile is stale

### Package Manager improvements
- Added explicit exit codes: RESOLUTION_FAILURE=1, CIRCULAR_DEPENDENCY=2, INVALID_MANIFEST=3, LOCKFILE_MISMATCH=4, GIT_CLONE_FAILURE=5

## Testing
- 16 new tests covering local deps install, lockfile format, reproducible installs, circular detection, verbose output, frozen lockfile, and exit codes

## VS Code Extension
- Version: 1.1.0
- Full language support with LSP + MCP dual-server architecture
- 14 LSP methods implemented
- MCP integration with automatic server lifecycle management
- 7 commands: Start/Stop/Restart MCP Server, Compile, Explain, Insert Example, Show Status
- 5 configuration settings for MCP server

## Diagnostics
- Full `ail check` coverage: Now runs complete compilation pipeline (forward-reference check → semantic analysis → type checking)
- CLI integration: `ail check` and `ail build` report identical diagnostics

## Known Issues
- 658 pre-existing ruff style issues (line-length, unused imports) - non-blocking
- 1 mypy duplicate-module issue in app tests - structural, outside compiler scope

## Upgrade Notes
- No breaking changes
- Lock file format has changed - existing `ail.lock` files will need regeneration via `ail install`
- Package.json version bumped to 1.1.0 for VS Code extension
